# STM32WL55 LoRa Prototype Handoff

## Current Goal
This project has been simplified from the original ST `SubGHz_Phy_PingPong_DualCore` example into a one-way STM32 transmitter for a REYAX LoRa receiver connected to an Arduino Uno.

The current firmware reads two analog channels on the STM32WL55 and transmits both raw ADC values in a REYAX-compatible frame.

## Current Radio Behavior
- One-way transmit only
- No RX / PingPong behavior intended for the prototype flow
- LoRa frequency: `915000000`
- LoRa preamble length: `4`
- Transmission period: `2000 ms`

The Uno / REYAX receiver should receive payloads in this form:

```text
R1:01234,R2:02345
```

Inside the full REYAX receive print this looks like:

```text
+RCV=1,17,R1:01234,R2:02345,<rssi>,<snr>
```

## ADC / Sensor Mapping
The firmware is now set up for the **final PCB mapping**, not the older Nucleo Arduino-header prototype mapping.

- Sensor 1 input: `PA0` -> `ADC_CHANNEL_0`
- Sensor 2 input: `PA1` -> `ADC_CHANNEL_1`

Important:
- This is one STM32 ADC peripheral sampling two channels sequentially.
- The current build is not targeting Nucleo Arduino `A0 = PB1` anymore.
- If testing on the Nucleo board again, make sure the voltage is applied to the actual MCU-connected pins used by this firmware: `PA0` and `PA1`.

## ADC Stability Choices
To make readings more stable:
- ADC sampling time was increased to `ADC_SAMPLETIME_160CYCLES_5`
- Each channel read discards the first conversion after switching channels
- Then it averages `4` samples

This logic lives in `CM4/SubGHz_Phy/App/subghz_phy_app.c`.

## Low-Power Note
Low power was disabled on purpose during debugging because the ADC only gave a valid reading after power-up and then fell to zero after Stop2/wake transitions.

Current setting:
- `CM4/Core/Inc/sys_conf.h`
- `LOW_POWER_DISABLE = 1`

This should be treated as a deliberate debug/prototype choice. If low power is needed later, the ADC wake/re-init path should be revalidated.

## Current Implementation Notes
- Radio sends in task/main context using the sequencer
- Timer is one-shot and re-armed after each send
- The frame is built as a REYAX-compatible header plus ASCII payload
- Payload length is currently `17`

## Known Caveats
- IDE linter errors around STM32 headers and HAL types were previously false positives in Cursor and did not necessarily indicate a real CubeIDE build error
- Because this project has been partly regenerated by CubeMX before, watch for CubeMX overwriting custom ADC pin/channel settings or radio parameters
- If CubeMX is used again, verify:
  - `PA0` and `PA1` are still configured as analog
  - channel configuration still uses `ADC_CHANNEL_0` and `ADC_CHANNEL_1`
  - radio frequency is still `915000000`
  - preamble is still `4`
  - low-power choice is still what you expect
